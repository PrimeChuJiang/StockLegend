> 本文档定义核心玩法循环、系统规则和数据结构，作为后续开发的基准。
> 第二轮设计讨论已闭合所有框架层开放问题，可直接进入编码。

# 第一部分：游戏循环

## 1. 游戏概述

### 1.1 一句话描述

你是一个无良财经自媒体人，通过撰写真假参半的文章操控舆论、影响股价，从中低买高卖赚取暴利。

### 1.2 核心体验

- **身份幻想**：幕后的市场操控者，用笔杆子撬动资本市场
- **策略深度**：素材选择 → 文章组合 → 发表时机 → 交易决策，环环相扣
- **玩家对抗**：发文操控 vs 打假反制，素材在进攻与防守间取舍
- **黑色幽默**：荒诞的市场事件、自我打脸的分析师、"编的比真的还准"的假新闻

### 1.3 核心循环

```
取材（抽牌）→ 撰写（组合素材卡）→ 发表/打假（操控舆论或反制对手）→ 交易（买卖股票获利）
```

### 1.4 胜利条件

固定 20 回合，打满结束。按**总资产（现金 + 持仓市值）排名**，排名最高者获胜。

- 纯排名制，无"目标金额"达标线
- 不设淘汰机制，破产玩家继续参与（可以打假搅局）
- 不提前结束

### 1.5 玩家资源

| 资源 | 说明 | 初始值 | 回合恢复 |
|------|------|-------|---------|
| **现金** | 用于买卖股票、付费推广、黑市交易 | 10000 | 无自动恢复 |
| **精力** | 每回合可撰写文章的次数 | 2 | 每回合重置为 2 |
| **人脉** | 用于窥探日程、定向取材、主流媒体发表 | 3 | 每回合 +1 |
| **信誉** | 影响文章效果和可用渠道 | 50 | 每回合向50的方向移动1（自然恢复） |

```
人脉的三种用途（互相竞争）：
  · 窥探日程事件 → 信息优势
  · 定向取材     → 素材优势
  · 主流媒体发稿 → 影响力优势
```

**资产总额** = 现金 + Σ(每只股票持仓数量 × 当前股价)

### 1.6 信息可见性

| 信息 | 可见性 |
|------|--------|
| 对手现金 | 公开 |
| 对手持仓 | 延迟公开（上一回合结束时的快照，当前回合实时变化不可见） |
| 对手手牌数量 | 公开（具体内容隐藏） |
| 已发表文章属性 | 公开（影响力、可信度、方向） |
| 对手信誉值 | 公开 |
| 黑市交易行为 | 匿名可见（"有人买/卖了一张卡"，不知道是谁） |

# 第二部分：游戏流程

## 2. 开局流程

```
开局流程：
  1. 玩家加入房间
  2. 随机确定选人顺序
  3. 按顺序依次选角色（公开，所有人可见，不可重复）
  4. 选角完成，每人抽 5 张初始素材卡
  5. 不提供初始建仓
  6. 进入第 1 回合 → MARKET_OPEN 公布第一周日程
```

## 3. 五日周期制

游戏以 **5 个回合为一个周期**（模拟一个交易周），总计 4 个周期共 20 回合。

每个周期第 1 天（第 1、6、11、16 回合），系统公布**本周期 5 天的新闻日程表**。日程表显示哪些天会发生什么类型的事件，但**不透露事件结果是利好还是利空**。

```
┌─ 第二周期日程公布（第6回合开始时）─────────────────────┐
│                                                        │
│  第 6天：（无预定事件）                                 │
│  第 7天：星海科技 · 季度财报发布                         │
│  第 8天：（无预定事件）                                 │
│  第 9天：央行 · 议息会议结果公布                         │
│  第10天：仁和医药 · 新药三期临床结果                     │
│                                                        │
│  ※ 事件结果未知，可消耗人脉提前窥探                      │
└────────────────────────────────────────────────────────┘
```

**关键时间线**：越早的事件留给玩家的准备窗口越短。

### 3.1 内幕窥探机制

```
窥探规则：
  · 目标：本周期内尚未到来的日程事件
  · 成本：1 人脉/次
  · 效果：立即揭示该事件的完整内容（利好/利空、影响力大小）
  · 时机：GATHER 阶段可执行窥探
  · 限制：每回合最多窥探 1 次
```

```
信息三层级：
  免费信息（日程公布时）：知道"第7天星海科技有财报事件"
  付费窥探（事件日前）：  花1人脉提前得知"财报暴雷，利空-4"
  自然揭示（事件当天）：  所有人看到完整结果
  突发新闻（完全未知）：  不在日程上，无法窥探
```

## 4. 单日回合阶段

每回合（每个交易日）按以下 **7 个阶段**顺序执行。

**行动模式**：GATHER / WRITING / TRADE 阶段所有玩家**同时行动**（提交后统一推进）；PUBLISH 阶段按**轮换顺序**依次行动（每回合起始玩家轮换）；MARKET_OPEN / MARKET_REACT / SETTLEMENT 为系统自动处理阶段。

```
┌─────────────────────────────────────────────────────────┐
│ 1. 开盘阶段 MARKET_OPEN                    【系统阶段】   │
│    · 若为周期第1天（5n+1），公布本周期5日日程表            │
│    · 揭示本回合的日程事件（如有），其结果此时自然公开        │
│    · 判定是否触发突发新闻（30%概率，从突发事件牌堆抽取）    │
│    · 所有环境牌效果生效，修改各股票情绪值                   │
├─────────────────────────────────────────────────────────┤
│ 2. 取材阶段 GATHER                         【同时行动】   │
│    · 从素材牌堆随机抽取 N 张（默认 3 张）                 │
│    · 可消耗人脉进行定向取材（见 6.3 节）                   │
│    · 可消耗人脉窥探未来日程事件（见 3.1 节）               │
│    · 可在情报黑市买卖素材卡（见 10 节）                    │
│    · 当前回合已揭示的环境事件加入"可引用素材池"            │
├─────────────────────────────────────────────────────────┤
│ 3. 撰写阶段 WRITING                        【同时行动】   │
│    · 从手牌中选择 2~4 张素材卡组合成文章                   │
│    · 系统实时计算文章属性（类型/影响力/可信度）             │
│    · 可撰写多篇文章，受精力值限制                          │
│    · 文章完成后进入"稿件区"，等待发表或继续压稿             │
├─────────────────────────────────────────────────────────┤
│ 4. 发表阶段 PUBLISH                        【轮换行动】   │
│    · 按轮换顺序依次行动，每个玩家轮到时可：                │
│      - 发表进攻性文章（操控股价），可多篇                  │
│      - 发表反驳文章（打假存活的历史文章），可多篇           │
│      - 不发表（压稿/跳过）                                │
│      - 以上可混合操作                                     │
│    · 选择发表渠道（自媒体/主流媒体/匿名论坛/付费推广）     │
│    · 发表的文章效果生效，修改对应股票的情绪值               │
│    · 反驳文章的合法目标：情绪修正器仍存活的所有历史文章     │
│    · 反驳文章结算详见第 9 节                               │
├─────────────────────────────────────────────────────────┤
│ 5. 市场反应阶段 MARKET_REACT               【系统阶段】   │
│    · 结算所有股票的最终情绪值                              │
│    · 根据情绪值计算各股票本回合涨跌幅，更新股价             │
├─────────────────────────────────────────────────────────┤
│ 6. 交易阶段 TRADE                          【同时行动】   │
│    · 玩家买入/卖出股票（按更新后的股价结算）                │
│    · 无限流动性模型，玩家对系统买卖，无抢购冲突             │
│    · 每回合交易次数有限（暂定 3 次）                       │
├─────────────────────────────────────────────────────────┤
│ 7. 收盘结算阶段 SETTLEMENT                 【系统阶段】   │
│    · 计算玩家当前资产总额（现金 + 持仓市值）                │
│    · 所有情绪修正器 remaining_turns -= 1，归零的移除        │
│    · 压稿区文章"新鲜度"衰减（归零则移入废稿区）             │
│    · 手牌超过上限（8张）的玩家强制弃牌到上限               │
│    · 若为第 20 回合，触发最终排名结算                      │
└─────────────────────────────────────────────────────────┘
```

> **备选方案：抢筹溢价**——如果 play test 发现交易层博弈太薄，可引入"同一回合多人买入同一只股票时成交价按买入人数上浮"（如 2 人同时买→×1.1，3 人→×1.2），模拟抢筹推高价格。

### 4.1 阶段枚举定义

```
Phase {
    MARKET_OPEN,    # 开盘（系统阶段）
    GATHER,         # 取材（同时行动）
    WRITING,        # 撰写（同时行动）
    PUBLISH,        # 发表 + 打假（轮换行动）
    MARKET_REACT,   # 市场反应（系统阶段）
    TRADE,          # 交易（同时行动）
    SETTLEMENT      # 收盘结算（系统阶段）
}
```

### 4.2 交易规则

- 每回合 TRADE 阶段可执行最多 **3 次**交易操作（买入或卖出各算一次）
- 买入数量以**手**为单位（1 手 = 100 股），最少买 1 手
- 卖出可任意数量（不限整手）
- **无限流动性模型**：玩家对系统买卖，无股票总量限制，无抢购冲突
- TRADE 阶段所有玩家**同时行动**
- 暂无做空机制（v1 简化，后续可扩展）
- 暂无手续费/税费（v1 简化）

```
Holding {
    stock_id: StringName    # 持有的股票
    quantity: int           # 持有数量
    avg_cost: float         # 平均持仓成本
}
```

### 4.3 手牌上限

- 素材卡手牌上限 **8 张**
- SETTLEMENT 阶段末尾，手牌超过上限的玩家**强制弃牌到上限**，玩家自选弃哪张
- 弃牌进入弃牌堆，不获得现金
- 角色之间上限暂时统一（后续可作为角色差异化手段）

# 第三部分：市场系统

## 5. 股票

游戏中包含若干只股票，分属不同行业。初版暂定：

| 股票名称 | 行业 | 初始股价 | 波动性 |
|---------|------|---------|-------|
| 星海科技 | 科技 TECH | 50 | 高 |
| 中原银行 | 金融 FINANCE | 80 | 低 |
| 绿源能源 | 能源 ENERGY | 40 | 中 |
| 仁和医药 | 医药 MEDICAL | 60 | 中 |

**波动系数**决定每点情绪值推动股价变动的绝对金额：

| 波动性 | 波动系数 | 含义 |
|--------|---------|------|
| 低 LOW | 0.5 | 每点情绪每回合推动股价 ±0.5 元 |
| 中 MEDIUM | 1.0 | 每点情绪每回合推动股价 ±1.0 元 |
| 高 HIGH | 1.5 | 每点情绪每回合推动股价 ±1.5 元 |

### 5.1 情绪值系统（修正器驱动）

每只股票的市场情绪值由**所有活跃的情绪修正器之和**决定，范围夹到 -50 ~ +50：

```
-50 ─────── -30 ─────── 0 ─────── +30 ─────── +50
极度恐慌     偏空       中性        偏多      极度乐观
```

```
当前情绪值 = clamp(Σ 所有活跃修正器的 value, -10, +10)
```

情绪修正器来自两个来源，格式统一：

```
SentimentModifier {
    source_id: StringName   # 来源标识（文章ID 或 环境牌ID）
    source_type: String     # "article" 或 "environment"
    target_stock: StringName
    value: int              # 修正值（正=利好，负=利空）
    remaining_turns: int    # 剩余持续回合数
}
```

**文章产生的修正器**：
- value = 文章影响力 × 方向（利好 +1 / 利空 -1）
- remaining_turns = 文章可信度（影响力决定幅度，可信度决定持续时间）

**环境牌产生的修正器**：
- value = 环境牌的 sentiment_modifier
- remaining_turns = 环境牌的 duration

**修正器生命周期**：
- MARKET_OPEN 阶段：环境牌修正器施加
- PUBLISH 阶段：文章修正器施加
- SETTLEMENT 阶段：所有修正器 remaining_turns -= 1，归零的修正器移除

没有任何活跃修正器时，情绪值为 0，股价不再变动（定格在当前位置）。

### 5.2 情绪值 → 股价变动（速度模型）

**情绪值控制股价变动的速度，而非股价本身。** 每回合 MARKET_REACT 阶段：

```
每回合股价变动 = 情绪值 × 波动系数
新股价 = 当前股价 + 情绪值 × 波动系数
```

核心特性：
- 情绪 > 0 → 股价持续上涨（涨速与情绪值成正比）
- 情绪 = 0 → 股价**定格在当前位置**，不会回弹到初始价
- 情绪 < 0 → 股价持续下跌

股价是**路径依赖**的——历史上所有情绪修正器的累积效果决定了当前价格。修正器到期只意味着"不再继续推动"，不意味着"之前的推动被撤销"。

### 5.3 股价与情绪的时序关系

**股价每回合只结算一次**，在 MARKET_REACT 阶段计算。回合内的时序：

```
MARKET_OPEN  → 环境牌修正器生效（情绪值变化）
GATHER       → 取材（情绪值不变）【同时行动】
WRITING      → 撰写（情绪值不变）【同时行动】
PUBLISH      → 文章/反驳文章修正器生效（情绪值变化）【轮换行动】
MARKET_REACT → 根据当前情绪值计算股价变动
TRADE        → 玩家按新股价买卖【同时行动】
SETTLEMENT   → 修正器 tick，到期的移除
```

关键设计：文章发表后至少生效一整回合才可能被打假（最快下一回合 PUBLISH 阶段才有人反驳）。当回合的股价变动不会被撤销，打假惩罚体现在后续回合的反转修正器。

典型操作流：

```
前几天 TRADE  → 提前建仓（买入目标股票）
当天 PUBLISH  → 发表利好文章，情绪修正器生效
当天 MARKET_REACT → 股价上涨
当天 TRADE    → 卖出获利
```

**操作顺序极其重要**——必须先建仓再发文，否则你自己把价格推高了再去买就亏了。

### 5.4 退市机制

当某只股票价格跌破**初始价的 20%** 时，触发退市：

| 股票 | 初始价 | 退市线 |
|------|-------|-------|
| 星海科技 | 50 | 10 |
| 中原银行 | 80 | 16 |
| 绿源能源 | 40 | 8 |
| 仁和医药 | 60 | 12 |

```
MARKET_REACT 阶段，股价结算后检查退市条件：
  1. 该股票标记为"已退市"
  2. 玩家该股票的所有持仓按退市价强制清算（通常亏损惨重）
  3. 该股票从游戏中移除：
     · 不再参与情绪/股价结算
     · 不再触发相关环境事件（日程中的未来事件取消）
     · 针对该股票行业的素材卡仍留在牌堆中，但若该行业已无上市股票则变为死牌
  4. 触发特殊事件文本："恭喜你，你用几篇文章把一家上市公司写没了"
```

### 5.5 股价变动完整示例

```
星海科技，初始价 50，波动系数 1.5，退市线 10

第6回合：
  MARKET_OPEN → 环境牌「芯片禁令」修正器 {value: -3, remaining: 2}
  PUBLISH     → 玩家利空文章修正器     {value: -8, remaining: 1}
  情绪值 = clamp(-3 + -8, -10, 10) = -10
  MARKET_REACT → 股价 = 50 + (-10 × 1.5) = 35
  SETTLEMENT  → 两个修正器 remaining 各 -1

第7回合：
  利空文章修正器到期移除
  芯片禁令修正器还剩 1 回合，value = -3
  情绪值 = -3
  MARKET_REACT → 股价 = 35 + (-3 × 1.5) = 30.5
  SETTLEMENT → 芯片禁令修正器 remaining -1

第8回合：
  芯片禁令修正器到期移除
  无活跃修正器，情绪值 = 0
  MARKET_REACT → 股价 = 30.5 + 0 = 30.5（定格，不会回到50）

第9回合：
  玩家发表利好文章修正器 {value: +5, remaining: 3}
  情绪值 = +5
  MARKET_REACT → 股价 = 30.5 + (5 × 1.5) = 38.0（开始回升，但靠的是新的推力）
```

设计意图：股价是所有历史情绪的累积结果。"推下去容易拉回来难"——砸盘只需要一波利空，但修复股价需要持续不断的利好推力。

## 6. 环境牌系统

### 6.1 环境牌层级

| 层级 | 影响范围 | 示例 |
|------|---------|------|
| **宏观事件 MACRO** | 所有股票 | 央行降息、全球衰退、贸易战 |
| **行业事件 INDUSTRY** | 某行业所有股票 | 芯片禁令、医药集采、新能源补贴 |
| **个股事件 COMPANY** | 单只股票 | CEO丑闻、财报暴雷、产品召回 |

### 6.2 抽取机制：五日周期日程 + 突发新闻

**新闻日程表（可预见）**

```
日程生成规则（暂定）：
  · 每个周期 2~3 个日程事件，随机分配到 5 天中
  · 日程事件只展示"事件标题"（如"星海科技 财报发布"），不展示结果
  · 事件结果（利好/利空/影响力大小）在事件当天 MARKET_OPEN 阶段揭示
  · 玩家可在事件日之前消耗人脉窥探结果（见 3.1 节）
```

**突发新闻（不可预见）**

```
突发新闻概率：30%（暂定）
触发时从"突发事件牌堆"随机抽取一张，立即生效
突发事件不在日程表上显示，无法被窥探
```

### 6.3 环境牌数据结构

```
EnvironmentCard {
    id: StringName
    name: String                   # 如"央行宣布降息50个基点"
    preview_name: String           # 日程表上显示的预告名
    tier: EventTier                # MACRO / INDUSTRY / COMPANY
    target_industry: Industry      # tier=INDUSTRY时指定行业；MACRO时为ALL
    target_stock: StringName       # tier=COMPANY时指定个股；否则为空
    sentiment_modifier: int        # 对目标情绪值的修正（如 +3 或 -4）
    duration: int                  # 持续回合数
    source_type: EventSourceType   # SCHEDULED（日程）或 BREAKING（突发）
    can_be_referenced: bool        # 玩家是否可引用为写作素材
    is_revealed: bool              # 运行时状态：结果是否已被揭示
    flavor_text: String
}
```

### 6.4 环境牌与文章的交互

**顺势/逆势修正**：

- 玩家文章方向与当前环境一致（顺势）→ 文章影响力 ×1.3
- 玩家文章方向与当前环境矛盾（逆势）→ 文章可信度 ×0.7

**引用环境事件为素材**：

- 当前回合已揭示的环境事件可被引用为写作素材
- 引用的环境事件视为一张**临时数据卡**（可信度 4，影响力 2，bias 与事件一致）
- 引用不消耗环境牌本身（多篇文章可引用同一事件）

# 第四部分：卡牌系统

## 7. 素材卡系统

### 7.1 素材卡类型

素材卡是玩家的核心手牌，代表可用于撰写文章的信息碎片。每张素材卡有三种用途（互斥）：

```
素材卡的三种用途（互斥消耗）：
  · 写文章 → 组合素材发表，影响股价
  · 卖到黑市 → 换取现金
  · 作为打假弹药 → 合成反驳文章攻击对手
```

| 类型 | 说明 | 可信度倾向 | 影响力倾向 |
|------|------|-----------|-----------|
| **数据卡 DATA** | 财报摘要、行业统计、K线图表 | 高（3~5） | 低（1~3） |
| **爆料卡 SCOOP** | 内幕消息、泄露文件、知情人爆料 | 中（2~4） | 高（3~5） |
| **谣言卡 RUMOR** | 小道消息、论坛传闻、"据知情人士透露" | 低（1~2） | 极高（4~5） |
| **观点卡 OPINION** | 专家评论、机构评级、分析师观点 | 中（2~4） | 中（2~4） |

### 7.2 素材卡属性

```
MaterialCard {
    id: StringName          # 唯一标识
    name: String            # 素材名称，如"星海科技Q3财报"
    type: MaterialType      # DATA / SCOOP / RUMOR / OPINION
    industry: Industry      # 所属行业标签（可为 MACRO 表示宏观通用）
    bias: Bias              # 倾向：BULLISH / BEARISH / NEUTRAL
    credibility: int        # 可信度 1~5
    impact: int             # 影响力 1~5
    tags: Array[StringName] # 语义标签，用于协同/冲突判定
    flavor_text: String     # 风味文字（黑色幽默描述）
}
```

### 7.3 取材机制：统一牌堆 + 定向抽取

所有素材卡属于**同一个牌堆**，不按类型或行业分开存放。

**普通取材（免费）**：每回合 GATHER 阶段自动随机抽取 3 张。

**定向取材（消耗人脉）**：

| 定向级别 | 人脉成本 | 效果 |
|---------|---------|------|
| 行业定向 | 1 人脉 | 指定一个行业，从牌堆中筛选出该行业的素材，随机抽 1 张 |
| 精确定向 | 2 人脉 | 指定行业 + 素材类型（如"科技行业的爆料卡"），筛选后随机抽 1 张 |

- 定向取材在普通取材之后执行，每回合可执行多次（受人脉限制）
- 若牌堆中没有符合条件的素材，人脉不消耗，取材失败

## 8. 写作方法卡系统

### 8.1 概述

写作方法卡代表玩家掌握的**写作技巧和手法**——不是你写什么，而是你怎么写。

```
两类卡牌的本质区别：
  素材卡 = 你写什么（信息、证据、谣言）→ 公共牌堆，抽取获得，用完消耗
  写作方法卡 = 你怎么写（技巧、角度、风格）→ 角色自带，永久持有
```

### 8.2 获取方式：角色自带

写作方法卡通过**角色选择**获得——每个角色自带 4 张专属写作方法卡。

```
获取规则：
  · 角色选定时，自动获得该角色的全部专属写作方法卡
  · 局内无法获取新的写作方法卡
  · 不同角色之间的写作方法卡可能有重叠
  · 所有角色额外持有 1 张通用基础卡「平铺直叙」（保底方法）
```

### 8.3 使用规则

```
· 每篇文章可附加 0~2 张写作方法卡
· 写作方法卡不消耗，使用后留在个人卡库
· 每张卡每回合最多使用 1 次
· 使用过的卡进入冷却，下回合不可使用
· 冷却持续 1 回合，之后自动恢复可用
```

### 8.4 数据结构

```
WritingMethodCard {
    id: StringName          # 唯一标识
    name: String            # 如"标题党"、"交叉验证"
    description: String     # 效果描述
    effect: Dictionary      # 效果参数
    rarity: Rarity          # COMMON / UNCOMMON / RARE
    flavor_text: String     # 黑色幽默描述
    is_on_cooldown: bool    # 运行时状态：是否在冷却中
}
```

### 8.5 写作方法卡一览

**基础类（影响数值）**：

| 卡牌名 | 稀有度 | 效果 |
|-------|-------|------|
| 平铺直叙 | 通用 | 无特殊效果（保底方法） |
| 标题党 | 普通 | 影响力 ×1.5，可信度 ×0.7 |
| 数据可视化 | 普通 | 文章中每张数据卡可信度 +1 |
| 情感渲染 | 普通 | 影响力 +3，可信度 -1 |
| 严谨论证 | 稀有 | 可信度 +2，影响力 -1 |
| 交叉验证 | 罕见 | 文章中每有两张不同类型素材，可信度 +1 |

**操控类（改变规则）**：

| 卡牌名 | 稀有度 | 效果 |
|-------|-------|------|
| 春秋笔法 | 稀有 | 可将 1 张素材的 bias 反转（利好↔利空） |
| 匿名化处理 | 普通 | 被打假时反驳方影响力要求 +3，可信度 -1 |
| 断章取义 | 稀有 | 可将数据卡或观点卡当作爆料卡使用 |
| 长期追踪 | 罕见 | 文章情绪修正器持续时间 +2 回合 |

**特殊类（独特能力）**：

| 卡牌名 | 稀有度 | 效果 |
|-------|-------|------|
| 病毒传播 | 罕见 | 文章同时影响目标行业的所有股票 |
| 连续剧 | 稀有 | 若上回合对同一股票发过文章，本篇影响力 ×2 |
| 危机公关 | 罕见 | 不产生新修正器，改为移除目标股票上一个己方修正器并反转 |
| 深喉网络 | 稀有 | 本回合定向取材不消耗人脉 |
| 震惊体 | 普通 | 影响力 ×1.5，可信度 ×0.5 |

# 第五部分：文章系统

## 9. 文章组合系统

### 9.1 组合规则

```
一篇文章 = 2~4 张素材卡 + 0~2 张写作方法卡
```

- 素材卡必须**行业标签一致**，或使用 MACRO（宏观）标签的万能素材
- 写作方法卡无行业限制，任何文章都能用
- 每回合撰写文章数量受**精力值**限制（每篇消耗 1 精力，每回合 2 精力）

### 9.2 文章属性计算

组合完成后，系统分三步计算文章属性：

**第一步：基础值（由素材决定）**

```
基础可信度 = floor(所有素材卡可信度的平均值)
基础影响力 = 所有素材卡影响力之和
文章方向   = 统计素材倾向多数决（利好多→利好，利空多→利空，平票→中性）
目标行业   = 素材共有的行业标签
```

**第二步：文章类型加成（由素材组合模式决定）**

| 组合模式 | 文章类型 | 特殊效果 |
|---------|---------|---------|
| 数据 + 数据 + 观点 | 深度研报 | 持续时间 +1 回合 |
| 爆料 + 数据 | 调查报道 | 可信度 +1 |
| 谣言 + 爆料 | 惊天独家 | 影响力 ×1.5，可信度 -1 |
| 谣言 + 谣言 | 阴谋论 | 50%概率影响力翻倍，50%概率文章无效 |
| 爆料 + 爆料 | 连环爆料 | 影响力 +3，可信度 -1 |
| 数据 + 观点 | 专家点评 | 可信度 +1 |
| 观点 + 观点 | 舆论造势 | 影响力 +2 |
| 其他组合 | 一般报道 | 无特殊效果 |

**第三步：写作方法卡修正**

依次应用附加的写作方法卡效果（加法先于乘法）。

```
最终可信度 = floor((基础可信度 + 类型加成 + 方法卡加成) × 方法卡乘数)
最终影响力 = floor((基础影响力 + 类型加成 + 方法卡加成) × 方法卡乘数)
```

示例：

```
2 张数据卡（可信度 4,3 影响力 2,2）+ 1 张观点卡（可信度 3 影响力 3）+ 「标题党」

第一步：基础可信度 = floor((4+3+3)/3) = 3，基础影响力 = 2+2+3 = 7
第二步：匹配"数据+数据+观点=深度研报"，持续时间+1
第三步：「标题党」影响力 ×1.5，可信度 ×0.7
最终：可信度 = floor(3 × 0.7) = 2，影响力 = floor(7 × 1.5) = 10
      持续时间 = 可信度2 + 类型加成1 = 3 回合
```

### 9.3 素材间的化学反应

**协同（Synergy）**——满足条件时提供额外加成：

| 条件 | 效果 | 设计意图 |
|------|------|---------|
| 两张素材 tags 有交集且来自不同 type | 可信度 +1（证据链） | 多源交叉验证更可信 |
| 爆料卡 + 数据卡指向同一结论（bias 一致） | 影响力 +2 | 有理有据的爆料更震撼 |
| 引用当前回合环境事件作为素材 | 可信度 +1（蹭热点） | 真实事件背书 |

**冲突（Contradiction）**——不良搭配导致惩罚：

| 条件 | 效果 | 设计意图 |
|------|------|---------|
| 同一篇文章中 bias 互相矛盾（既利好又利空） | 影响力 ×0.5 | 自相矛盾的文章没人信 |
| 多张谣言卡堆叠（≥2） | 可信度额外 -1 每张 | 防止无脑堆谣言 |
| 高可信度素材（≥4）与低可信度素材（≤2）混用 | 可信度按低者计算 | 一颗老鼠屎坏一锅粥 |

## 10. 发表系统

### 10.1 发表渠道

| 渠道 | 成本 | 影响力修正 | 可信度修正 | 打假防御 | 信誉交互 | 解锁条件 |
|------|------|-----------|-----------|---------|---------|---------|
| 自媒体 | 免费 | ×0.7 | ×0.8 | 无 | 正常涨跌 | 初始可用 |
| 主流财经媒体 | 1 人脉 | ×1.0 | ×1.2 | 反驳方需额外+3影响力 | 正常涨跌 | 信誉 ≥ 60 |
| 匿名论坛 | 免费 | ×0.5 | ×0.5 | 反驳方影响力要求减半 | 不涨不跌信誉 | 初始可用 |
| 付费推广 | 500 现金 | ×1.3 | ×0.9 | 无 | 被打假信誉-20（翻倍） | 初始可用 |

渠道定位：
- **自媒体**：保底选择，中庸
- **主流媒体**：贵+有门槛，但数值最好且防打假
- **匿名论坛**：免费脏活渠道，效果弱但不赌信誉
- **付费推广**：花钱买影响力，但被锤代价很大

**渠道冷却**：被玩家打假成功后，自媒体和主流媒体冷却 1 回合。匿名论坛和付费推广不冷却。

### 10.2 压稿机制

- 撰写完成的文章可以不立即发表，留在"稿件区"
- 每回合 SETTLEMENT 阶段，压稿的文章**新鲜度 -1**
- 新鲜度归零后文章自动移入废稿区（素材丢失）
- 初始新鲜度暂定为 3（可压稿 3 回合）

### 10.3 废稿

- 玩家可主动废弃文章，素材卡进入弃牌堆
- 不消耗精力，但损失素材

## 11. 文章打假系统

### 11.1 概述

打假就是写文章——玩家通过撰写一篇**反驳文章**来攻击对手已发表的文章。这是纯玩家驱动的对抗机制（无系统自动事实核查）。

```
文章目标类型：
  · 目标是股票 → 影响股价情绪（进攻性文章）
  · 目标是对手的文章 → 与对手文章进行影响力比对（反驳文章）
```

### 11.2 打假时机与流程

PUBLISH 阶段按轮换顺序依次行动，每个玩家轮到时可以选择发表进攻性文章、反驳文章，或两者混合。

**反驳目标范围**：所有情绪修正器仍存活（remaining_turns > 0）的历史文章均可被反驳。修正器到期后文章不再是合法目标。

```
打假流程：
  1. 轮到你的 PUBLISH 回合时，选择一篇存活的对手文章作为反驳目标
  2. 用手牌素材合成反驳文章（与普通文章相同的合成流程）
  3. 反驳文章必须与被打假文章立场相反
  4. 立即结算打假结果
```

关键设计：文章发表后至少生效一整回合才可能被打假。高可信度文章持续时间长，暴露在打假窗口的回合也更多——可信度是双刃剑。

### 11.3 打假结算

```
情况一：打假成功（反驳文章影响力 > 被打假文章影响力）
  → 被打假文章的情绪修改器立即移除
  → 施加反转修正器：value = 原修正值反向 × 0.5，remaining = 2 回合
  → 被打假玩家信誉 -10（付费推广渠道翻倍 -20；匿名论坛不扣）
  → 被打假文章的自媒体/主流媒体渠道进入冷却 1 回合

情况二：打假失败（反驳文章影响力 ≤ 被打假文章影响力）
  → 被打假文章影响力减去反驳方影响力，修正器的 value 相应降低
  → 反驳文章消耗掉，不产生任何效果（素材白费）
```

注：渠道对打假的影响：
- 被打假文章通过**主流媒体**发表 → 反驳方需额外 +3 影响力
- 被打假文章通过**匿名论坛**发表 → 反驳方影响力要求减半

### 11.4 完整示例

```
第5回合：
  PUBLISH → A 通过自媒体发表利好文章，影响力 10，目标星海科技
           修正器 {value: +10, remaining: 3}
  MARKET_REACT → 星海科技情绪 +10，股价上涨

第6回合：
  PUBLISH → 轮到 B，选择反驳 A 的文章
           B 合成利空反驳文章，影响力 12
           12 > 10 → 打假成功！
           · A 的修正器移除
           · 施加反转修正器 {value: -5, remaining: 2}
           · A 信誉 -10，自媒体渠道冷却 1 回合
  MARKET_REACT → 星海科技情绪 -5，股价下跌
```

### 11.5 打假的战略深度

**攻击方（打假者）的决策：**
- 我手里的素材是写文章赚钱，还是留着打假别人？
- PUBLISH 轮换顺序中，我排在对手后面可以看到对手发了什么再决定
- 我现在不打，等对手继续加仓再打，他的损失更大——但修正器可能先到期

**防守方（被打假者）的决策：**
- 选什么渠道？主流媒体有 +3 防御但要花人脉；匿名论坛效果弱但不扣信誉
- 低可信度文章持续时间短，可能在被打假之前就过期——"打一枪就跑"

**旁观者的决策：**
- 两个人在打架，我要不要趁机在他们忽略的股票上操作？
- 我要不要加入打假方，趁机打压这只股票然后低价买入？

## 12. 信誉系统

> **注：信誉系统为 play test 验证项。** 如果测试证明冗余，备选方案：砍掉信誉倍率，主流媒体改为纯花人脉解锁，邪典效应改为角色固有被动。

玩家有一个持续追踪的**信誉值**，范围 0~100，初始 50：

| 信誉区间 | 称号 | 效果 |
|---------|------|------|
| 80~100 | 权威财经媒体人 | 文章影响力 ×1.3，解锁主流媒体渠道 |
| 60~79 | 知名财经博主 | 文章影响力 ×1.1，解锁主流媒体渠道 |
| 40~59 | 普通自媒体 | 无加成无惩罚 |
| 20~39 | 争议性写手 | 文章可信度 ×0.8，主流媒体渠道关闭 |
| 0~19 | 阴谋论博主 | 主流媒体关闭，解锁"邪典效应"：阴谋论文章影响力 ×2 |

信誉值变动：
- 发表**高可信度**文章且未被打假 → +3
- 被玩家打假成功 → -10（付费推广渠道翻倍为 -20；匿名论坛不扣）
- 发表的文章预测与实际走势一致 → +5（"事后诸葛亮"加成）
- 每回合自然恢复 +1（公众遗忘）

## 13. 情报黑市系统

### 13.1 概述

情报黑市是一个系统做市商驱动的素材卡交易市场。玩家可以在 GATHER 阶段在黑市买入或卖出素材卡。

### 13.2 定价机制

```
定价公式：
  基准价 = 稀有度阶梯价
  黑市售价（玩家买入）= 基准价 + 随机值
  玩家卖出价 = 基准价 - 随机值
```

| 稀有度 | 基准价 | 买入价范围 | 卖出价范围 |
|-------|--------|-----------|-----------|
| 普通 COMMON | 200 | 220~300 | 100~180 |
| 稀有 UNCOMMON | 500 | 550~700 | 300~450 |
| 罕见 RARE | 1000 | 1100~1400 | 600~900 |

### 13.3 黑市库存

```
系统上架规则：
  · 每回合系统最多上架 3 张素材卡
  · 稀有度概率：普通 60% / 稀有 30% / 罕见 10%
  · 素材类型、行业、立场随机
  · 库存上限 5 张，满了系统停止注入
```

### 13.4 信息展示

```
可见信息：素材类型、关联行业、价格
不可见信息：立场（利好/利空）、具体数值（可信度/影响力）
```

### 13.5 交易规则

```
· 买入素材卡：消耗现金，不消耗行动值
· 卖出素材卡：获得现金，不消耗行动值
· 每回合买卖次数：暂不限制
· 时机：GATHER 阶段
· 匿名性：其他玩家只能看到"有人买/卖了一张卡"，不知道是谁
```

# 第六部分：角色系统

## 14. 角色系统

### 14.1 概述

每位玩家在游戏开始前选择一个角色。角色决定了玩家的**被动技能、主动技能、专属写作方法卡和初始资源**。

- 每局游戏中同一角色只能被一位玩家选择（不可重复）
- 初版提供 6 个角色，支持 2~6 人游戏
- 角色选择公开，所有人可见

### 14.2 角色数据结构

```
Character {
    id: StringName
    name: String
    title: String             # 头衔/身份描述
    passive_skill: Skill      # 被动技能（全局生效，无需激活）
    active_skill: Skill       # 主动技能（消耗资源，每回合限用一次）
    writing_methods: Array[WritingMethodCard]  # 专属写作方法卡（4张）
    starting_cash: int
    starting_reputation: int
    starting_connections: int
    flavor_text: String
}

Skill {
    id: StringName
    name: String
    description: String
    cost: Dictionary          # 消耗资源（被动技能为空）
    cooldown: int
    is_passive: bool
}
```

## 15. 黑色幽默元素

### 15.1 反讽事件（特殊环境牌）

| 事件名称 | 效果 |
|---------|------|
| "你编的假消息居然成真了" | 最近一篇被打假的文章效果恢复，信誉 +5 |
| "认真写的分析被当成软文" | 最近一篇高可信度文章影响力减半 |
| "竞争对手抄袭你的假新闻" | 对应股票情绪效果再次触发（免费加倍） |
| "监管约谈但调查员也在炒股" | 收到罚款通知但罚款金额以股票形式支付给你 |
| "AI 生成的分析报告满天飞" | 本回合所有文章影响力 ×0.5（信息过载） |
| "股评人跳槽当了脱口秀演员" | 随机一位"专家"观点卡从牌堆中移除 |

### 15.2 卡牌风味文字示例

**素材卡**：
- 谣言卡「据不愿透露姓名的保洁阿姨称」：可信度 1，影响力 4
- 数据卡「第三方审计报告（第47页脚注）」：可信度 5，影响力 1
- 爆料卡「CEO的前妻在朋友圈发的截图」：可信度 2，影响力 5
- 观点卡「从未看对过的首席分析师最新研判」：可信度 3，影响力 3

**写作方法卡**：
- 「震惊体」：你的文章标题必须以感叹号结尾！！！（影响力 ×1.5，可信度 ×0.5）
- 「AI代笔」：一分钟出稿，质量随缘（精力消耗减半，可信度 -2）
- 「学术黑话」：没人看得懂但显得很专业（可信度 +2，影响力 -1）

## 17. 待定数值（play test 调整）

以下数值需要通过原型测试来调整：

1. **数值平衡**：素材卡的可信度/影响力范围、波动系数、初始资金等
2. **做空机制**：当前只支持买入持有，后续可扩展
3. **多局 Meta 进程**：单局结束后是否有跨局解锁内容
4. **素材卡数量**：初步估算每种类型 8~12 张，共约 32~48 张
5. **环境牌数量**：日程事件和突发事件各需要多少张
6. **角色平衡**：6 个角色之间的强度平衡
7. **黑市数值**：基准价、随机范围、系统上架频率
8. **信誉系统**：是否保留（见第 12 节备注）
